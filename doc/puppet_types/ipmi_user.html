Resource Type: ipmi_user

Defined in:
lib/puppet/type/ipmi_user.rb

Summary

Manages BMC user accounts via IPMI.

Overview

Supports both ipmitool and freeipmi backends. Manages user name, password, privilege level, enabled state, SOL access, and channel access in an idempotent fashion.

Examples:

Create an admin user

ipmi_user { 'admin_user':
  user     => 'admin',
  password => Sensitive('s3cret'),
  user_id  => 3,
  priv     => 4,
  channel  => 1,
  enable   => true,
}

Disable a user

ipmi_user { 'old_user':
  user_id => 5,
  channel => 1,
  enable  => false,
}

Properties

  • enable (defaults to: true)

    Whether this user account should be enabled or disabled.

    Supported values:
    • true
    • false
  • priv (defaults to: 4) 
      Privilege level for the user:

    4 - ADMINISTRATOR 3 - OPERATOR 2 - USER 1 - CALLBACK

Parameters

  • bmcconfig_cmd (defaults to: /usr/sbin/bmc-config)

    Path to the bmc-config binary (freeipmi).

  • channel (defaults to: 1)

    The IPMI channel number for user access configuration. Defaults to 1.

  • ipmitool_cmd (defaults to: /usr/bin/ipmitool)

    Path to the ipmitool binary.

  • name (namevar)

    Resource title (arbitrary label for this user resource).

  • password

    Password for the IPMI user. May be a Sensitive value. Required when enable is true.

  • purge_id_mismatch (defaults to: false)

    When true, any IPMI user slot that holds the given username at an ID other than user_id will be blanked and disabled before the desired slot is configured. Only applies when enable is true.

    Supported values:
    • true
    • false
  • user (defaults to: root)

    The IPMI username to set.

  • user_id (defaults to: 3)

    The numeric IPMI user slot ID. On SuperMicro IPMI, user id 2 is reserved for the ADMIN username. On ASUS IPMI, user id 2 is reserved for the admin username.
